# Output Formats: Gap Analysis & Improvement Modes

PHASE 4 (Gap Analysis) 및 PHASE 5 (Apply Improvements)에서 사용한다.

---

## PHASE 4: Gap Analysis Card

```
================================================================
  GAP ANALYSIS
  Your draft vs [Blueprint source] | Section: [section type]
================================================================

Well-aligned:
  - [dimension]: [what's working]
  - [dimension]: [what's working]

Needs improvement:
  | Dimension          | Your draft              | Blueprint standard           | Example fix                  |
  |--------------------|-------------------------|------------------------------|------------------------------|
  | Tone               | [observed issue]        | [target behavior]            | "[original]" -> "[revision]" |
  | Logical flow       | [observed issue]        | [target behavior]            | [structural note]            |
  | Transitions        | [observed issue]        | [target behavior]            | "[original]" -> "[revision]" |
  | Vocabulary         | [observed issue]        | [target behavior]            | "[original]" -> "[revision]" |

Similarity scores (out of 10):
  Tone:               [X]/10
  Sentence structure: [X]/10
  Logical flow:       [X]/10
  Transitions:        [X]/10
  Vocabulary:         [X]/10
  Citation style:     [X]/10
  -------------------------
  Overall:            [X]/10
```

### Scoring Guide

| Score | Meaning | Recommended Mode |
|-------|---------|------------------|
| 8-10 | Already matches Blueprint well | Mode A (targeted edits) |
| 5-7 | Noticeable gaps, targeted edits sufficient | Mode A |
| 1-4 | Significant divergence, full rewrite recommended | Mode B |

After scoring, recommend a mode:
- Overall >= 7 -> suggest Mode A (targeted edits)
- Overall < 7 -> suggest Mode B (full rewrite)

사용자가 이미 Mode를 선택했으면 점수에 관계없이 해당 Mode로 진행.

---

## PHASE 5 Mode A: Sentence-Level Feedback

초고를 순서대로 처리한다. 모든 단락을 다룬다.
문제 문장마다:

```
[ORIGINAL]
"[exact original text]"

[REVISION]
"[improved version]"

[RATIONALE]
- [which Blueprint dimension was applied + specific rule, in 1-2 lines]
```

Rules:
- 잘 쓰인 문장은 간략히 확인만 하고 넘어간다
- 수정 불필요한 문장은 리라이팅하지 않는다
- Gap Analysis에서 가장 낮은 점수의 차원부터 우선 처리
- 각 수정에 적용된 Blueprint 차원 번호를 명시한다 (e.g., "Dim 1: Tone")

---

## PHASE 5 Mode B: Full Rewrite

전체 입력 섹션을 리라이팅한다. 6개 Blueprint 차원 모두 적용.

### Preservation Rules (보존 원칙)
- 원문의 모든 주장(argument) 보존
- 모든 데이터 포인트 보존 (수치, 단위, 불확도)
- 모든 인용(citation) 보존
- Figure/Table 참조 번호 유지
- 과학적 의미 변경 절대 금지
- 스타일만 변환한다

### 리라이팅 후 Change Summary

```
-----------------------------------------
CHANGE SUMMARY
-----------------------------------------
| Change type            | Count | Example                                     |
|------------------------|-------|---------------------------------------------|
| Tone adjustment        | N     | Removed informal phrasing; added hedging    |
| Verb substitution      | N     | "shows" -> "demonstrates"                   |
| Sentence restructure   | N     | Simple -> "Although X, Y" complex form      |
| Transition added       | N     | Inserted "Furthermore", "Notably"           |
| Logic reordering       | N     | Reordered to Claim -> Evidence -> Interp.   |
| Nominalization applied | N     | "X increased" -> "the increase in X"        |
-----------------------------------------
Arguments preserved: YES  |  Data preserved: YES  |  Citations preserved: YES
```

### Post-Rewrite Verification

리라이팅 완료 후 자체 검증:
1. 원문의 모든 주장이 리라이팅에 존재하는지 확인
2. 모든 인용이 보존되었는지 확인
3. 데이터 수치가 변경되지 않았는지 확인
4. Blueprint 6개 차원이 모두 반영되었는지 확인
5. 자연스러운 학술 문체로 읽히는지 확인 (기계적 치환 아닌지)
